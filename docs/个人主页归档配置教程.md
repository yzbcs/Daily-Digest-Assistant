# 个人主页每日推送归档配置教程

本教程教你如何搭建一个**每日论文推送归档系统**，效果类似于：https://yzbcs.github.io/study/

---

```
┌─────────────────────────────────────────────────────────────┐
│                    Daily-Digest-Assistant                    │
│                   (GitHub Actions)                          │
│                         │                                   │
│              每天自动运行 main.py                            │
│                         │                                   │
│              生成 preview.html                               │
│                         │                                   │

│                         │                                   │

                          │

│                         │                                   │

└─────────────────────────────────────────────────────────────┘
```

## 第一部分：创建个人主页 (GitHub Pages)

### 1.1 初始化仓库

1. 在 GitHub 创建新仓库，命名为 `{你的用户名}.github.io`
   - 例如：`yzbcs.github.io`

2. 克隆到本地：
   ```bash
   git clone https://github.com/你的用户名/你的用户名.github.io.git
   cd 你的用户名.github.io
   ```

### 1.2 使用 Academic Pages 模板

本项目使用 [Academic Pages](https://github.com/academicpages/academicpages.github.io) 模板：

1. 下载模板：
   ```bash
   git clone https://github.com/academicpages/academicpages.github.io.git template
   ```

2. 复制模板内容（**不要复制 `.git` 文件夹**）：
   ```bash
   rm -rf template/.git
   cp -r template/* .
   rm -rf template
   ```

3. 提交初始模板：
   ```bash
   git add .
   git commit -m "Initial template"
   git push origin master
   ```

### 1.3 配置个人主页

编辑 `_config.yml`：
```yaml
title: "你的名字"
name: "你的名字"
description: "个人主页描述"

author:
  name: "你的名字"
  bio: "一句话简介"
  github: "你的GitHub用户名"
  avatar: "https://avatars.githubusercontent.com/你的GitHub用户名"

url: "https://你的用户名.github.io"
repository: "你的用户名/你的用户名.github.io"
```

### 1.4 创建归档目录

```bash
mkdir -p study/daily
```

创建 `study/index.html`（归档展示页面）：
```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>📬 Daily Digest · 归档</title>
    <style>
        /* 样式代码（参考本仓库的 study/index.html） */
    </style>
</head>
<body>
    <!-- 归档页面结构（参考本仓库的 study/index.html） -->
</body>
</html>
```

创建 `study/data.json`（初始索引）：
```json
{"notes": []}
```

### 1.5 提交归档基础结构

```bash
git add study/
git commit -m "Add study archive structure"
git push origin master
```

## 第二部分：创建 Daily-Digest-Assistant 项目

### 2.1 创建仓库

在 GitHub 创建新仓库：`Daily-Digest-Assistant`

克隆到本地：
```bash
git clone https://github.com/你的用户名/Daily-Digest-Assistant.git
cd Daily-Digest-Assistant
```

### 2.2 项目结构

```
Daily-Digest-Assistant/
├── .github/
│   └── workflows/
│       └── daily.yml          # GitHub Actions 配置
├── study/
│   ├── index.html              # 归档页面模板
│   └── data.json               # 归档索引
├── scripts/
│   └── update_study_data.py   # 更新 data.json 的脚本
├── templates/
│   └── study_index.html        # 归档页面模板副本
├── main.py                      # 主程序
└── requirements.txt             # Python 依赖
```

### 2.3 核心文件

#### `scripts/update_study_data.py`

```python
#!/usr/bin/env python3
"""更新 study/data.json 文件，扫描所有 HTML 文件并生成索引"""

import os
import json
from pathlib import Path

def update_data_json():
    # 获取所有 HTML 文件
    notes = []
    daily_dir = Path("study/daily")

    if daily_dir.exists():
        for html_file in sorted(daily_dir.rglob("*.html"), reverse=True):
            # 从文件名提取日期（如 2026-04-08）
            date_str = html_file.stem  # "2026-04-08"

            notes.append({
                "id": date_str,
                "title": f"📬 Daily Digest · {date_str}",
                "path": str(html_file),  # 如 "daily/2026/04/2026-04-08.html"
                "created": date_str,
                "tags": ["Daily Digest", "arXiv", "小红书"]
            })

    # 写入 data.json
    with open("study/data.json", "w", encoding="utf-8") as f:
        json.dump({"notes": notes}, f, ensure_ascii=False, indent=2)

    print(f"Updated data.json with {len(notes)} entries")

if __name__ == "__main__":
    update_data_json()
```

### 2.4 GitHub Actions 配置

创建 `.github/workflows/daily.yml`：

```yaml
name: Daily Paper Digest

on:
  schedule:
    - cron: "0 4 * * *"    # UTC 04:00 = 北京时间 12:00
  workflow_dispatch:        # 支持手动触发

jobs:
  send-digest:
    runs-on: ubuntu-latest
    env:
      FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: "true"

    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "22"

      - name: Install Node.js dependencies
        run: npm install

      - name: Install Python dependencies
        run: pip install -r requirements.txt

      - name: Run daily digest
        env:
          LLM_API_KEY: ${{ secrets.LLM_API_KEY }}
          EMAIL_USER:  ${{ secrets.EMAIL_USER }}
          EMAIL_PASS:  ${{ secrets.EMAIL_PASS }}
          EMAIL_TO:    ${{ secrets.EMAIL_TO }}
          XHS_COOKIE:  ${{ secrets.XHS_COOKIE }}
        run: python main.py

      - name: Upload preview.html artifact
        uses: actions/upload-artifact@v4
        with:
          name: digest-html
          path: preview.html

  sync-pages:
    if: github.repository == '你的用户名/Daily-Digest-Assistant'
    needs: send-digest
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Download preview.html artifact
        uses: actions/download-artifact@v4
        with:
          name: digest-html

      - name: Prepare digest HTML
        run: |
          DATE=$(date +%Y-%m-%d)
          MONTH=$(date +%Y/%m)
          mkdir -p study/daily/$MONTH
          cp preview.html study/daily/$MONTH/$DATE.html

      - name: Copy index.html
        run: cp templates/study_index.html study/index.html

      - name: Update data.json
        run: python3 scripts/update_study_data.py

      - name: Push to GitHub Pages
        run: |
          git config --global user.name "github-actions[bot]"
          git config --global user.email "github-actions[bot]@users.noreply.github.com"

          # Clone 目标仓库（替换为你的用户名）
          git clone --depth=1 https://x-access-token:${{ secrets.YZBCS_TOKEN }}@github.com/你的用户名/你的用户名.github.io.git target-repo

          # 复制 study 文件夹到目标仓库
          cp -r study/* target-repo/study/

          # 推送
          cd target-repo
          git add study/
          git diff --quiet && git diff --staged --quiet || git commit -m "auto: sync daily digest $(date +%Y-%m-%d)"
          git push origin master
```

## 第三部分：配置 Secrets

在 Daily-Digest-Assistant 仓库的 `Settings > Secrets and variables > Actions` 中添加：

| Secret 名称 | 说明 |
|------------|------|
| `YZBCS_TOKEN` | GitHub Personal Access Token（需要 repo 和 workflow 权限） |
| `LLM_API_KEY` | LLM API 密钥（如 OpenAI） |
| `EMAIL_USER` | 发送邮箱 |
| `EMAIL_PASS` | 邮箱授权码 |
| `EMAIL_TO` | 接收邮箱 |
| `XHS_COOKIE` | 小红书 Cookie（可选） |

### 创建 Personal Access Token

1. 进入 GitHub `Settings > Developer settings > Personal access tokens`
2. 点击 `Generate new token (classic)`
3. 勾选 `repo` 和 `workflow` 权限
4. 生成并复制 Token

## 第五部分：自定义修改

### 5.1 修改归档页面样式

编辑 `templates/study_index.html` 中的 `<style>` 部分。

### 5.2 修改论文来源

编辑 `main.py` 中的 `main()` 函数，调整 fetcher 配置。

### 5.3 修改推送时间

编辑 `.github/workflows/daily.yml` 中的 cron 表达式：
```yaml
schedule:
  - cron: "0 4 * * *"    # 修改这里的数字
```

**北京时间对照**：
- `0 4 * * *` = 北京时间 12:00
- `0 2 * * *` = 北京时间 10:00
- `0 0 * * *` = 北京时间 08:00

## 常见问题

### Q: 归档页面显示"加载中..."怎么办？

1. 检查浏览器控制台错误
2. 确认 `data.json` 可以正常访问
3. 检查 HTML 文件路径是否正确

### Q: GitHub Actions 推送失败？

1. 确认 `YZBCS_TOKEN` 权限正确
2. 确认目标仓库名称正确
3. 检查 Actions 日志

### Q: 如何删除某天的归档？

1. 删除对应的 HTML 文件
2. 本地运行 `python scripts/update_study_data.py` 更新索引
3. 提交并推送
