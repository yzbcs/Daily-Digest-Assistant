自定义 LLM 配置指南

← 返回更新日志

📖 概述

从 v1.1.0 开始，你可以通过 config.yml 的 custom_llm 配置项接入任意 LLM API，无需修改代码。

只要你的 API 是 OpenAI 兼容 或 Anthropic 兼容 协议，都可以直接接入。

⚙️ 配置格式

llm_provider: my_api

custom_llm:
  my_api:
    sdk: openai
    base_url: "https://your-api-endpoint.com/v1"
    model: "your-model-name"
字段 必填 说明
sdk 是 "openai" 或 "anthropic"
base_url 是 API 地址，注意要包含 /v1 后缀（OpenAI 协议）
model 是 模型名称，请向你的 API 提供商确认

📝 配置示例

示例 1：OpenAI 官方 API

llm_provider: my_openai

custom_llm:
  my_openai:
    sdk: openai
    base_url: "https://api.openai.com/v1"
    model: "gpt-4o-mini"

示例 2：DeepSeek API

llm_provider: deepseek

custom_llm:
  deepseek:
    sdk: openai
    base_url: "https://api.deepseek.com/v1"
    model: "deepseek-chat"
注意：DeepSeek 已内置在 v1.1.0+ 中，这里仅作为示例。

示例 3：Anthropic Claude 官方 API

llm_provider: my_claude

custom_llm:
  my_claude:
    sdk: anthropic
    base_url: "https://api.anthropic.com"
    model: "claude-3-haiku-20240307"

🔑 API Key 配置

API Key 统一填写在 GitHub Secrets 的 LLM_API_KEY 中，程序会自动使用。

路径：仓库 → Settings → Secrets and variables → Actions → New repository secret

❓ 常见问题

  • Q: 我的 API 地址没有 /v1 后缀怎么办？
    A: 大多数 OpenAI 兼容 API 都需要 /v1，请向你的提供商确认正确的地址。
  • Q: 配置后报 "不支持的 llm_provider" 错误？
    A: 检查 llm_provider 的值是否与 custom_llm 下的名称完全一致。
  • Q: 可以同时配置多个自定义提供商吗？
    A: 可以！在 custom_llm 下添加多个条目，通过修改 llm_provider 切换。
